twistedActor: Version History

1.1.0b1 2015-10-14

  • Add a generic parser. Not polished

1.0.7 2015-06-04

  • CommandQueue.addCmd: if the provided command does not have a cmdVerb then provide one named "dummy".
  • testUtils.startLogging: if the log directory does not exist then create it.

1.0.6 2015-03-10

  • Remove Device.startCmdList
  • Change DeviceSet.startCmdDict to call Device.startCmd for each command in the list of commands; this means each Device must be capable of handling many commands in a row, e.g. by using a CommandQueue if queuing is required.
  • expandUserCmd is now available as a free function.

1.0.5 2015-02-05

  • Bug fix: DeviceSet.startCmd and startCmdDict did not wait for all device commands to finish if a device had more than one command.

Documentation update 2015-01-13

  • Added a software license (BSD-3 in doc/license.txt) and a readme file (ReadMe.md).

1.0.4 2015-01-07

  • Improved error message when a linked command (instance of LinkCommands) fails.
  • DeviceSet: simplified and rationalized the handling of callback functions in the startCmd methods:
    • The value returned by the callback is ignored, instead of being used to start a new device command. This was a feature that was not being used and greatly complicated the code.
    • Failure of a callback result in a traceback, but does not result in failure of the command.
    • Improved the error reported if a device command fails.
  • Bug fix: DeviceSet.startCmd ignored the timeLim argument.
  • Bug fix: DeviceSet.expandSlotList ignored the connOnly argument.

1.0.3 2014-12-16

  • Added "showReplies" arguments to device commands.

1.0.2 2014-10-31

  • DeviceActor: added a new property "model".
  • DeviceSet: added a new argument "slot" to _addDevCallbacks and _removeDevCallbacks.
  • Actor: the default help command now strips a leading "!" from the doc string, since that is used as a signal to Doxygen.
  • ActorDevice: added property "model".

1.0.1 2014-10-30

  • Improve the state messages in DevCmdVar resulting from failed CmdVar callback.
  • Added from __future__ import print_function and updated print statements accordingly.
  • Made the doc strings compatible with doxygen (""" -> """!) and fixed the errors reported by Doxygen.

1.0.0 2014-10-28

  • Bug fix: Device's connection state callback did not detect if a connection failed or was failing.
  • Bug fix: FileLogger would needlessly reject basePath if it did not include a directory.

0.9.4 2014-10-21

  • Overhauled DeviceSet.replaceDev to make its behavior safer and more predictable:
    • Initialize the new device.
    • Disconnect the replaced device, instead of merely sending INIT.
    • Fixed a bug whereby DeviceSet would attempt to send a status command after disconnecting.
    • Replace the device even if connecting and initializing the new device fails.

0.9.3 2014-09-24

  • Modernized the exception catching syntax.

0.9.2 2014-08-29

  • Standardized capitalization of keywords output by TwistedActor to camelCase with leading lowercase. Most started with an initial capital letter.
  • Enabled generation of Doxygen documentation.
  • Added link to Doxygen documentation in main manual and created this version history file.

0.9.1 2014-08-27

This version requires RO package 3.5.3.

  • BaseActor now refuses to start, with a suitable error message, if the user port is in use.
  • Bug fix: DispatcherWrapper.queueCmd did not notice a command failure if the specified callback codes did not include failure codes.

0.9.0 2014-08-29

First version with version history.
